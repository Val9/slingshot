Neudesic Slingshot

Slingshot provides a HTML/Mobile Web client and library for Microsoft SharePoint Server 2010. Using jQuery Mobile, Slingshot makes it easy to expose tasks, appointments, documents, and other objects from any SharePoint 2010 instance to a mobile device.

You can even access features of your device, such as taking a picture on your phone, and uploading it to a document library.

Quick Start

  1. Clone the repository, 'git clone https://github.com/neudesic/slingshot.git'

  2. Modify the compat.browser file in SharePoint (often found in c:\inetpub\wwwroot\wss\VirtualDirectories\80\App_Browsers). Locate the sections that apply to your devices, and set the isMobileDevice value to false. An example of how to do this for the iPhone can be found in the Support folder.

  3. Create a new virtual directory (e.g. slingshot) under the SharePoint route in Inetmgr, and add the index.html, Scripts, and Styles folders.

Browse to http://yourserver/slingshot from your mobile device!

FAQs

Q. What devices does Slingshot support today?

Slingshot supports all devices supported by jQuery Mobile (iOS, Android, Blackberry, Windows Phone 7).

Q. How does authentication work with Slingshot?

Exactly the same as SharePoint does today (default is either NTLM or Basic auth). We do have plans to support cached credentials and additional options.

Q. Does Slingshot support any device-specific functionality?

Yes, using the Slingshot Cordova wrapper project, you can upload photos from the device to a SharePoint document library.

Q. What are the deployment options for Slingshot?

By default, you access Slingshot through your mobile browser, however by using the Slingshot Cordova wrapper project, you can also create a native application that wraps the mobile content.

Q. Does that mean I can deploy my SharePoint mobile app to the AppStore?

Technically, yes, however Apple's policy is to commonly reject internal, employee facing applications. A more suitable approach might be to consider Apple's Enterprise Program.
